---
title: "Crypto Addresses"
description: "Registering customer-owned crypto wallet addresses as onramp settlement destinations."
---

A crypto address is the customer's own wallet — where converted crypto is sent after an onramp deposit settles.

Call [`POST /crypto-address/add`](/api-reference/crypto-address/add-customers-crypto-address) <Icon icon="link" size={14} />:

```json
{
  "customerId": "e2f8c1d4-3b7a-4c19-9f6e-1a2b3c4d5e6f",
  "currency": "USDT",
  "address": "0xA1b2C3d4E5f6789012345678901234567890aBcD",
  "label": "Main Wallet",
  "network": "ETH"
}
```

Crypto addresses have **no approval step** — there is no `status` field on this resource. The address is usable immediately after creation.

See [`GET /crypto-address/list`](/api-reference/crypto-address/get-customers-crypto-address-list) <Icon icon="link" size={14} /> and [`GET /crypto-address/detail`](/api-reference/crypto-address/get-customers-crypto-address-detail) <Icon icon="link" size={14} /> to look up existing addresses.
