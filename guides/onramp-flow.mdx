## Use Case

Enable customers to onramp **fiat into cryptocurrency** (`USDT`, `USDC`). This works by spinning up a virtual account tied to an autoramp rule that automatically converts and forwards incoming fiat to a crypto address.

Examples:

- Create an **AUD virtual account** which converts to USDT on TRX for wallet address 1
- Create a **EUR virtual account** which converts to USDC on Ethereum for wallet address 2

## Example Flow

<Steps>
  <Step title="Set up the flow">
    Create an onramp autoramp flow for a customer, linking a new AUD virtual account to the customer's USDT wallet on TRX.
  </Step>
  <Step title="Customer sends fiat">
    Customer transfers AUD 1,000 into the virtual account via bank transfer or PayID.
  </Step>
  <Step title="Coinut converts">
    Coinut detects the deposit, confirms it, and executes the trade automatically.
  </Step>
  <Step title="Crypto arrives">
    ~998 USDT is delivered to the customer's registered wallet.
  </Step>
  <Step title="Repeatable">
    The customer can deposit again at any time — the virtual account and flow are persistent.
  </Step>
</Steps>

## Prerequisites

Every step must complete before moving to the next.

<Steps>
  <Step title="Customer is CONFIRMED">
    The customer must have completed KYC. See [Customer Lifecycle](/customers/customer-lifecycle).
  </Step>
  <Step title="Register the destination wallet">
    Register the customer's crypto wallet via [Crypto Addresses](/addresses/crypto-addresses). The `to` field in the onramp flow must match an address already registered for this customer.
  </Step>
  <Step title="(Optional) Preview the rate">
    Call [`GET /autoramp-flow/trade-preview`](/api-reference/autoramp-flow/trade-preview) <Icon icon="link" size={14} /> with `fromCurrency`, `toCurrency`, and `amount` for an indicative rate. This is a live estimate, not a locked quote — the flow always executes at the rate current when the deposit clears.
  </Step>
</Steps>

## Implementation

Create the onramp flow with [`POST /autoramp-flow/onramp`](/api-reference/autoramp-flow/create-an-onramp-flow-with-virtual-account) <Icon icon="link" size={14} />. The virtual account is created automatically from `from`.

### Response

```json
{
  "status": "SUCCESS",
  "data": {
    "id": "uuid-12345"
  }
}
```

The response only contains the new flow's `id`. Fetch [`GET /autoramp-flow/detail`](/api-reference/autoramp-flow/get-autoramp-flow-detail) <Icon icon="link" size={14} /> to retrieve the created virtual account's banking details (`payId`, `accountNumber`, `iban`, `bicSwift`) under `fromVirtualAccount` — display these to the customer.

### Error response

```json
{
  "status": "INVALID_PARAMETERS",
  "message": "invalid parameters"
}
```

## Monitor via Webhooks

Subscribe to `DEPOSIT_RECEIVED`, `DEPOSIT_APPROVED`, `TRADE_CREATED`, and `TRADE_SETTLED` rather than polling. See [Webhooks](/concepts/webhooks), and [Trades](/transactions/trades) for on-chain verification once `TRADE_SETTLED` fires.
