This page walks through the full onramp journey end to end, at a glance — from creating a customer to confirming on-chain settlement. Each step links to the page with full details. Offramp follows the mirror sequence — see [Offramp Flow](/guides/offramp-flow).

The journey breaks into three phases: **customer onboarding**, **destinations & deposit rails**, and **monitoring & settlement**.

## Phase 1: Customer Onboarding

<Steps>
  <Step title="Create Customer" icon="user-plus">
    <Card>
      Call [`POST /customer/create`](/api-reference/customer/create-a-new-customer) with the customer's name, email, and `externalId`.

      ```json
      {
        "name": "John Doe",
        "email": "john@example.com",
        "type": "DEPOSITOR",
        "isIndividual": true,
        "externalId": "user_12345"
      }
      ```

      See [Customer Lifecycle](/customers/customer-lifecycle) for the full field reference.
    </Card>
  </Step>
  <Step title="Wait for KYC Approval" icon="shield-check">
    <Card>
      Listen for the `CUSTOMER_APPROVED` webhook event. You can also poll [`GET /customer/detail`](/api-reference/customer/get-customer-details) to check status.

      Customer status transitions: `PENDING` → `CONFIRMED` | `REJECTED` (a customer can also end up `DELETED`). See [Customer Lifecycle](/customers/customer-lifecycle).
    </Card>
  </Step>
</Steps>

## Phase 2: Destinations & Deposit Rails

<Steps>
  <Step title="Add Crypto Address" icon="wallet">
    <Card>
      Call [`POST /crypto-address/add`](/api-reference/crypto-address/add-customers-crypto-address) to register the customer's wallet as the onramp destination. Store the returned `id` — you'll need it for the onramp flow. See [Crypto Addresses](/addresses/crypto-addresses).
    </Card>
  </Step>
  <Step title="Create Onramp Flow" icon="arrows-rotate">
    <Card>
      Call [`POST /autoramp-flow/onramp`](/api-reference/autoramp-flow/create-an-onramp-flow-with-virtual-account) with the fiat source and crypto destination:

      ```json
      {
        "from": {
          "customerId": "cust_abc123",
          "currency": "AUD"
        },
        "to": "crypto_address_id_from_step_3"
      }
      ```

      This creates a virtual account automatically and links its deposits to automatic crypto conversion. The response only contains the new flow's `id`. See [Onramp Flow](/guides/onramp-flow) for full details.
    </Card>
  </Step>
  <Step title="Display Virtual Account to Customer" icon="building-columns">
    <Card>
      Fetch [`GET /autoramp-flow/detail`](/api-reference/autoramp-flow/get-autoramp-flow-detail) and read the virtual account's banking details from `fromVirtualAccount`:

      | Currency | Key Fields |
      | --- | --- |
      | AUD | `payId`, `bicSwift` (BSB), `accountNumber` |
      | EUR / GBP / SGD / USD | `iban`, `bicSwift` (SWIFT/BIC) |

      See [Virtual Accounts](/virtual-accounts/virtual-accounts-guide) for the full field reference.
    </Card>
  </Step>
</Steps>

## Phase 3: Monitoring & Settlement

<Steps>
  <Step title="Monitor Deposits" icon="arrow-down-to-line">
    <Card>
      Listen for `DEPOSIT_RECEIVED` (funds detected) and `DEPOSIT_APPROVED` (funds cleared), or poll deposits with the customer ID. See [Deposits](/transactions/deposits).
    </Card>
  </Step>
  <Step title="Track Trade" icon="chart-line">
    <Card>
      After deposit approval, listen for `TRADE_CREATED` (trade initiated) and `TRADE_SETTLED` (trade complete, crypto sent). See [Trades](/transactions/trades).
    </Card>
  </Step>
  <Step title="Confirm Settlement" icon="circle-check">
    <Card>
      Once the crypto has been broadcast, the `TRADE_SETTLED` payload includes a nested `withdrawal.txId` — use that to verify the transaction on-chain:

      | Network | Explorer |
      | --- | --- |
      | ERC-20 | `https://etherscan.io/tx/{withdrawal.txId}` |
      | TRC-20 | `https://tronscan.org/#/transaction/{withdrawal.txId}` |

      `withdrawal` may be absent on the first delivery of `TRADE_SETTLED` if the broadcast hasn't completed yet — treat it as optional. See [Trades](/transactions/trades).
    </Card>
  </Step>
</Steps>
