---
title: "IP Whitelisting"
description: "Restrict API access by source IP address."
---

An IP whitelist restricts which source IPs are allowed to call the API with your credentials, as an additional layer on top of HMAC request signing (see [Authentication](/getting-started/authentication)).

## Get the Current Whitelist

Call [`GET /ip-white-list`](/api-reference/common/get-ip-whitelist) <Icon icon="link" size={14} />:

```json
{
  "status": "SUCCESS",
  "data": {
    "ipWhiteList": ["115.236.134.118", "47.89.58.120"]
  }
}
```

## Update the Whitelist

Call [`POST /ip-white-list`](/api-reference/common/update-ip-whitelist) <Icon icon="link" size={14} /> with the full replacement list:

```json
{
  "ipWhiteList": ["115.236.134.118", "47.89.58.120"]
}
```

```json Response
{
  "status": "SUCCESS",
  "data": {
    "message": "IP whitelist updated successfully"
  }
}
```

<Warning>
  `ipWhiteList` is a **full replacement**, not an incremental add — the array you send becomes the entire whitelist. Include every IP you want to keep allowed, not just the one you're adding.
</Warning>

<Note>
  An **empty array** (`"ipWhiteList": []`) means **no IP restriction** — any source IP can call the API as long as requests are correctly signed. This is also the default for a newly onboarded account.
</Note>

## Error Response

```json
{
  "status": "INVALID_PARAMETERS",
  "message": "invalid parameters"
}
```

<Note>
  IP whitelisting is a network-level restriction on top of, not a replacement for, request signing. Every request still requires a valid HMAC-SHA256 signature regardless of whitelist configuration. Webhook deliveries from Coinut are **not** subject to your IP whitelist.
</Note>
